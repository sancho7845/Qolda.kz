# Qolda.kz - Өзара көмек көрсету және әлеуметтік қолдау платформасы 🤝

**Qolda.kz** — еріктілер (волонтерлер) мен көмекке мұқтаж жандарды (ұйымдастырушылар, қарттар, т.б.) байланыстыратын заманауи және толықтай дайын (production-ready) әлеуметтік платформа. 

---

## 🚀 Негізгі мүмкіндіктер мен функциялар

1. **🔒 Тіркелу / Кіру және Аутентификация**:
   - Firebase Auth негізінде жасалған қауіпсіз кіру/шығу және тіркелу жүйесі.
   - Әрбір қолданушының рөлі бар: **Ерікті (Volunteer)**, **Ұйымдастырушы (Organizer/Seeker)**, немесе **Администратор (Admin)**.
   - 6 түрлі түрлі-түсті иллюстрациялық аватарлар мен аты-жөні арқылы бірегей профильді орнату.

2. **📋 Тапсырмалармен (Көмек сұранымдарымен) жұмыс**:
   - **Жаңа тапсырма қосу**: Сипаттамасын, басымдығын (шұғыл, орташа, төмен), санатын (Қарттарға көмек, Медициналық көмек, Техникалық қолдау және т.б.) таңдау.
   - **Мекенжай және Геолокация**: Жүйе браузер арқылы орналасқан жерді автоматты түрде анықтайды (GPS координаталары арқылы) немесе мекенжайды қолмен енгізуге мүмкіндік береді.
   - **Картамен байланыс**: Мекенжайды Google Maps арқылы бір батырмамен жылдам ашып көру мүмкіндігі.
   - **Сурет/Файл жүктеу**: Әрбір көмек сұранымына растайтын фотосурет немесе құжат жүктеуге болады.

3. **🔄 Көмек көрсету кезеңдері (Lifecycle)**:
   - Еріктілер тапсырманы "Қабылдау" батырмасын басып, өз міндетіне алады.
   - Тапсырма орындалғаннан кейін көмек сұраушы оны "Аяқталды" деп растайды.
   - Орындалғаннан соң бір-біріне 1-5 жұлдызды бағалау және пікір (Review) қалдыру тетігі іске қосылады.

4. **👤 Жеке бөлме (Профиль)**:
   - Статистиканы бақылау (Орындалған тапсырмалар саны, рейтинг, ағымдағы белсенді көмектер).
   - Еріктілер басқалардан алған кепілдемелер мен пікірлер жиынтығын көре алады.

5. **👑 Админ Панель**:
   - Толық платформа статистикасы (барлық тапсырмалар, еріктілер мен қолданушылар статистикасы).
   - Жалған/дұрыс емес тапсырмаларды бір батырмамен тез арада жою.
   - Ереже бұзған пайдаланушыларды уақытша немесе біржола бұғаттау (Ban user).
   - Шағымдарды (Complaints/Reports) қарау және шешу.

---

## 🛠 Технологиялық стек

- **Frontend**: React 18+ (TypeScript), Vite, Tailwind CSS, Lucide React (Иконкалар).
- **Backend**: Firebase Auth, Cloud Firestore (Нақты уақыттағы деректер алмасу - Real-time subscriptions).
- **Animations**: Framer Motion (интерактивті ауысулар мен визуализациялар).
- **SPA Routing**: Vercel-мен үйлесімді бағыттау контурлары.

## ⚡️ Vercel-ге Тегін Deployment жасау жолы (Step-by-Step)

Qolda.kz жобасы **Vercel** хостингіне толығымен бейімделген және `vercel.json` SPA бағыттау конфигурациясы алдын ала қосылған.

### 1-қадам. Жобаны GitHub-қа жүктеңіз
Егер сіз AI Studio арқылы жүктеп алсаңыз немесе экспорттасаңыз, локальді каталогта мына командаларды орындаңыз:
```bash
git init
git add .
git commit -m "feat: Qolda.kz production ready release"
# GitHub-та жаңа репозиторий ашып, соған байланыстырыңыз:
git remote add origin YOUR_GITHUB_REPO_URL
git branch -M main
git push -u origin main
```

### 2-қадам. Vercel-де жаңа жоба қосыңыз
1. [Vercel Dashboard](https://vercel.com)-қа өтіп, **"Add New" -> "Project"** таңдаңыз.
2. Өзіңіздің GitHub аккаунтыңызды жалғап, `Qolda.kz` репозиторийін **Import** түймесімен таңдаңыз.

### 3-қадам. Қоршаған орта айнымалыларын (`Environment Variables`) қосу
Жоба сіздің жеке Firebase базаңызбен жұмыс істеуі үшін Vercel орнату терезесінде **"Environment Variables"** бөліміне өтіп, қалтаңыздағы `.env.example` үлгісіне сәйкес мына айнымалыларды енгізіңіз:

| Environment Variable Name | Сипаттамасы (Мәні) |
| :--- | :--- |
| `VITE_FIREBASE_PROJECT_ID` | Сіздің Firebase жобаңыздың ID-і |
| `VITE_FIREBASE_APP_ID` | Firebase Web App ID-і |
| `VITE_FIREBASE_API_KEY` | Firebase API кілті |
| `VITE_FIREBASE_AUTH_DOMAIN` | Firebase Auth Domain (мысалы, `.firebaseapp.com`) |
| `VITE_FIREBASE_FIRESTORE_DATABASE_ID` | Деректер базасының ID-і (әдетте `(default)`) |
| `VITE_FIREBASE_STORAGE_BUCKET` | Файлдар сақтау орнының URL мекенжайы |
| `VITE_FIREBASE_MESSAGING_SENDER_ID` | Хабарлама жіберушінің ID-і (цифрлық код) |

> ⚠️ **Маңызды**: Егер бұл айнымалыларды Vercel интерфейсінде көрсетпесеңіз, алаңдамаңыз! Жоба автоматты түрде біздің алдын ала бапталған өндірістік (production-configured) Firebase жобасына қосылып, **іске бірден толық жұмыс істеп кіріседі** (zero-config).

### 4-қадам. "Deploy" басыңыз
Орнату аяқталғаннан кейін **Deploy** батырмасын басыңыз. Vercel сізге 1 минут ішінде өндірістік және қауіпсіз `https://...vercel.app` мекенжайын сыйлайды!

## 🔒 Деректер қауіпсіздігі (Firestore Security Rules)

Платформаның барлық деректері Firebase Firestore қауіпсіздік ережелерімен қорғалған. Толық сынақтан өткен заманауи ережелер құжаты `firestore.rules` файлында орналасқан. Оның негізгі қорғаныс тіректері:

- **Қауіпсіздік торы (Safety Net)**: рұқсат етілмеген кез келген оқу және жазу сұранымдары әуел бастан бұғатталған.
- **Пайдаланушы деректері**: тек тіркелген қолданушылар ғана өз профилін жасай алады. Пайдаланушы рөлін (мысалы, `isAdmin` немесе `isBanned` статустарын) өз бетімен ауыстыра алмайды, оған тек рұқсаты бар Админ құқылы.
- **Тапсырмалар**: кез келген адам кез келген тапсырманы өзегерте алмайды. Ерікті тек "жаңа" тапсырманы қабылдай алады немесе одан бас тарта алады, ал жұмыстың аяқталуын тек тапсырманы жариялаған иесі (Creator) ғана растай алады.
- **Бағалау және Шағымдар**: ешкім өзі-өзіне пікір қалдыра алмайды. Шағымдар (Reports) базасын оқуға тек жүйелік Админдердің құқығы бар.
