# Qolda.kz - Өзара көмек көрсету және әлеуметтік қолдау платформасы 🤝

**Qolda.kz** — еріктілер (волонтерлер) мен көмекке мұқтаж жандарды (жәрдем іздеушілер, қарттар, мүмкіндігі шектеулі жандар) белсенді байланыстыратын заманауи және толықтай дайын (production-ready) әлеуметтік платформа. Бұл қосымша нақты уақыттағы деректермен және интерактивті картамен жұмыс істейді.

---

## 🚀 Негізгі мүмкіндіктер мен функциялар

1. **🔒 Тіркелу және Аутентификация**:
   - Firebase Auth негізінде жасалған қауіпсіз тіркелу және кіру жүйесі.
   - Қолданушының рөлі: **Ерікті (Volunteer)** немесе **Ұйымдастырушы (Organizer/Seeker)**.
   - Поштаны растау (Email Verification) жүйесі. Расталмаған пайдаланушыларға қауіпсіздік мақсатында тапсырма қосу немесе қабылдау әрекеттері бұғатталады.
   - Суретті аватарлар мен аты-жөнді қамтитын бірегей профильді толтыру мүмкіндігі.

2. **📋 Тапсырмалар (Көмек сұранымдары)**:
   - **Тапсырма қосу**: Санаты (Қарттарға көмек, Босату/Тасымалдау, Оқу-ағарту, Техникалық қолдау және т.б.), басымдығы (Шұғыл, Орташа, Төмен) және мерзімі (Deadline).
   - **Мекенжай және Геолокация**: Браузер GPS геолокациясын немесе Алматы қаласы бойынша мекенжайды автоматты түрде анықтау және картада көрсету.
   - **Файл немесе Сурет жүктеу**: Тапсырмаға қатысты қажетті сурет немесе құжатты тікелей Firebase Storage-ға қосу мүмкіндігі.

3. **🗺 Интерактивті Картамен жұмыс**:
   - Тапсырмалар карта бетінде көрнекі маркерлер түрінде бейнеленеді. 
   - Мерзімі өткен (Expired), бұғатталған (Blocked) немесе аяқталған тапсырмалар негізгі тізімдер мен картадан автоматты түрде сүзіледі (көрсетілмейді).

4. **🔄 Көмек көрсетудің толық процесі**:
   - Еріктілер тапсырманы "Қабылдау" батырмасы арқылы өз жауапкершілігіне алады.
   - Орындалған соң, Сұраныс иесі келісімді растап, орындаушыға 1-5 жұлдызды бағалау және пікір қалдыра алады.
   - Екі тарап бір-біріне бағалау (Reviews) бере алады.

5. **👑 Серпінді Админ Панелі**:
   - Платформа бойынша толық статистика.
   - Шағымдарды (Complaints/Reports) басқару.
   - Күдікті немесе заңсыз тапсырмаларды бір батырмамен өшіру.
   - Ереже бұзған пайдаланушыларды уақытша немесе біржола бұғаттау (Ban user). Бұғатталған пайдаланушыларға кез келген белсенді әрекеттер жабылады.

## 🛠 Технологиялық стек

- **Frontend**: React 18+ (TypeScript), Vite, Tailwind CSS, Lucide React (Иконкалар).
- **Батарея**: Firebase Auth, Cloud Firestore (Real-time subscriptions), Firebase Storage.
- **Анимация**: Framer Motion (жұмсақ өтуші эффектілер мен UI үйлесімділіктері).
- **SPA конфигурациясы**: `vercel.json` бағыттауы.

## 💻 Локальді орнату және іске қосу

Локальді компьютерде жобаны іске қосу үшін мына қадамдарды орындаңыз:

### 1-қадам. Репозиторийді жүктеп алу және бумаларды орнату
```bash
# Жоба каталогына өтіңіз
cd qolda-kz

# Тәуелділіктерді (dependencies) орнатыңыз
npm install
```

### 2-қадам. Қоршаған ортаны баптау (`.env`)
Жоба қалтасында `.env.local` немесе `.env` файлын жасап, өзіңіздің Firebase деректеріңізді енгізіңіз (мына айнымалылар `.env.example` файлында көрсетілген):
```env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project_id.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project_id.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_FIREBASE_FIRESTORE_DATABASE_ID=(default)
```

### 3-қадам. Әзірлеу серверін іске қосу
```bash
npm run dev
```
Осыдан кейін қосымша локальді желіде `http://localhost:3000` (немесе бапталған кез келген портта) ашылады.

## 🔒 Firebase қызметтерін баптау

Қосымша дұрыс жұмыс істеуі үшін Firebase консолінде төмендегі баптауларды орындау қажет:

1. **Authentication**: 
   - **Email/Password** кіру тәсілін іске қосыңыз.
2. **Cloud Firestore Database**:
   - Жаңа деректер қорын жасаңыз. `(default)` ретінде баптаңыз.
3. **Storage**:
   - Файлдарды сәтті жүктеу үшін Firebase сақтау орнын (Storage) іске қосыңыз.
4. **Қауіпсіздік ережелері (Rules Deployment)**:
   - Жоба түбіріндегі `firestore.rules` және `storage.rules` ережелерін заңдастырыңыз. Олар пайдаланушылардың рөлдерін, бұғатталуларын және электрондық поштасының расталғанын қатаң тексереді.

## 🚀 Vercel Хостингіне орнату (Deploy)

Жобаның `vercel.json` конфигурациялық файлы толық жарамды. Vercel-ге орналастыру үшін:

1. Жобаңызды GitHub репозиторийіне жүктеңіз.
2. [Vercel](https://vercel.com) Dashboard-қа өтіп, репозиторийді импорттаңыз.
3. Орнату терісінде (Configuration) жоғарыда жазылған `VITE_FIREBASE_*` айнымалыларын **Environment Variables** бөліміне қосыңыз.
4. **Deploy** батырмасын басыңыз! Сіздің қосымшаңыз 1 минут ішінде іске қосылады.
